# Соглашения по ведению документации

Этот документ устанавливает единые стандарты для создания, организации и обновления документации проекта.

## Правила наименования файлов

Все файлы документации должны соответствовать следующему формату:

```
UPPER-CASE-KEBAB-CASE.md
```

**Примеры:**
- `PROJECT-OVERVIEW.md`
- `DATABASE-SCHEMA.md`
- `API-REFERENCE.md`
- `DEPLOYMENT-GUIDE.md`

## Правила версионирования документов

Версионирование — по **дате последнего изменения**. Не вводим отдельный семвер (1.0, 1.1) — это избыточно и всегда устаревает.

```
Дата: YYYY-MM-DD
```

**Пример:**
```
Дата: 2026-04-15
```

Обновляйте дату каждый раз, когда вносите существенные изменения в документ. Поле `Версия: X.Y` не использовать.

## Правила перекрестного ссылания

Используйте **относительные ссылки** для связи между документами:

```markdown
Смотрите [DATABASE-SCHEMA.md](../DATABASE-SCHEMA.md)
Дополнительно: [API-REFERENCE.md](./API-REFERENCE.md)
```

**Правила:**
- Ссылайтесь относительно текущей директории
- Используйте `../` для подъёма на уровень выше
- Не используйте абсолютные пути

## Шаблон заголовка документа

Каждый документ начинается с YAML-метаблока:

```markdown
---
Проект: [PROJECT_NAME]
Дата: YYYY-MM-DD
Статус: Draft | Review | Approved | Archived
Автор: [AUTHOR]
---

# Название документа

[Основной контент]
```

**Важно:** названия **файлов** — в `UPPER-CASE-KEBAB-CASE.md` (`DATABASE-SCHEMA.md`), названия **папок** — в `lower-case-kebab-case` с числовым префиксом для сортировки (`00-meta`, `04-technical`). Это не опечатка, а сознательный выбор: папки сортируются визуально по префиксам, файлы внутри — выделяются регистром.

## Передача документации AI-помощникам

Когда нужно передать документацию Claude или ChatGPT для работы:

### 1. Подготовка

- Четко опишите задачу, которую должен выполнить AI
- Укажите контекст: для какого проекта, какая цель
- Приложите релевантные документы в полном виде

### 2. Форма передачи

```
Контекст проекта:
- Название: [PROJECT_NAME]
- Описание: [КРАТКОЕ_ОПИСАНИЕ]
- Релевантные документы: [СПИСОК_ФАЙЛОВ]

Задача:
[ПОДРОБНОЕ_ОПИСАНИЕ_ТОГО_ЧТО_НУЖНО_СДЕЛАТЬ]

Материалы:
[ПОЛНЫЙ_ТЕКСТ_ДОКУМЕНТОВ_ИЛИ_ССЫЛКИ]
```

### 3. Указания для AI

- Указывайте формат ответа (markdown, JSON, код и т.д.)
- Просите придерживаться стиля и структуры существующих документов
- Напомните о соглашениях проекта (правила наименования, версионирования)
- Попросите включать мета-информацию в новые документы

### 4. Проверка результатов

- Проверьте соответствие соглашениям
- Обновите версию и дату последнего обновления
- Убедитесь в корректности относительных ссылок
- Перед коммитом пересмотрите содержание на предмет точности
