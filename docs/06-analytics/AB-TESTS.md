## Советы по тестированию

### Ошибки и как их избежать

| Ошибка | Описание | Решение |
|---|---|---|
| **Пиковое наблюдение** | Смотреть результаты до завершения | Решить длительность заранее |
| **Multiple Testing** | Смотреть много метрик | Выбрать 1-2 основные |
| **Seasonal effects** | Сезонность влияет на результат | Тестировать достаточно долго |
| **Network effects** | Юзеры влияют друг на друга | Рандомизировать по user_id |
| **Small sample** | Слишком мало юзеров | Увеличить длительность теста |

### Статистическая мощность (Power)

```
Power = Вероятность найти effect, если он существует
        (обычно 80% или 90%)

Пример:
- Если effect size = 10%
- Power = 80%
- Significance = 95%
- То нужен sample = XXX users
```
