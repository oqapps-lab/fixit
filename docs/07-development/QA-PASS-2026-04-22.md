## Bugs found

### BLOCKER

**B1 — Processing screen crashes** (`children.toUpperCase is not a function` after fix B2; originally `Element type invalid: CheckGlyph undefined`).
- `CheckGlyph` imported from `components/ui/NoirGlyphs` в [processing.tsx:17](app/(onboarding)/processing.tsx:17) и [paywall/success.tsx](app/paywall/success.tsx), но не экспортирован из `NoirGlyphs.tsx`.
- STATUS.md уже отмечал это как blocker ("Glyphs consolidation"), но тяжесть недооценена — **crash на середине onboarding**, юзер застревает.
- **Фикс применён:** добавил `CheckGlyph` в [components/ui/NoirGlyphs.tsx:23](components/ui/NoirGlyphs.tsx:23).
- **TODO:** также добавить `BookmarkGlyph / ShieldGlyph / DocumentGlyph` (упоминаются в STATUS.md).

**B2 — `children.toUpperCase()` crash в `Label` / `DocRef`** при undefined children.
- [components/ui/Label.tsx:41](components/ui/Label.tsx:41) и [components/ui/DocRef.tsx:40](components/ui/DocRef.tsx:40) слепо вызывают `children.toUpperCase()`. TypeScript типизирует `children: string`, но runtime допускает undefined (например при `{variableThatMayBeUndefined}`).
- Краш наблюдался после тапа "DIY Guide" в repair detail — вылетел BlueprintRoofScene subtree.
- **Фикс применён:** `String(children ?? '').toUpperCase()` в обоих файлах. Это 2-строчная защита от класса crash'ей.
- **TODO root cause:** найти точку, где передаётся undefined, и поправить вызывающий код.

### HIGH

**H1 — Location step: Continue CTA скрыт под клавиатурой**
- [app/(onboarding)/location.tsx](app/(onboarding)/location.tsx): `justifyContent: 'space-between'` внутри фиксированного View + `autoFocus` на `TextInput` → клавиатура открывается сразу и перекрывает CTA.
- Юзер физически не может продолжить, пока не скроется клавиатура. Скроется только через hardware Return (в sim) — но у number-pad нет кнопки Return на реальном iPhone.
- **Фикс:** обернуть контент в `<KeyboardAvoidingView behavior="padding" />` ИЛИ слушать `onChangeText` и автоматически фокусить Continue после ввода 5 цифр, ИЛИ перенести CTA внутрь InputAccessoryView (iOS).

### MEDIUM

**M1 — Option-cards без accessibilityRole="button"** на Context screen (Never tried / Some experience / Done this before / Budget / Mid-range / Premium) и Fix-Selection screen (DIY / Hybrid / Pro).
- `mobile_list_elements_on_screen` показывает их как чистый StaticText — VoiceOver не доберётся, Detox / Maestro не смогут их тапнуть по labels.
- Нарушение правила 11 из ui-qa protocol.
- **Фикс:** обернуть option-карточки в Pressable с `accessibilityRole="button"` и `accessibilityLabel={option.title + ' ' + option.subtitle}`.

**M2 — Status bar overlap** на welcome screen при скролле.
- Контент скроллится под системный status bar без blur/fade оверлея. На первом экране "Snap a photo..." + "In sixty seconds" видны сквозь "12:00" и wifi-иконки.
- `StatusBar style="light" translucent` в root layout, но NoirScreen не накладывает gradient/blur сверху при scroll.
- **Фикс:** добавить top-gradient overlay в NoirScreen (absolute, height ≈ insets.top+10, linear-gradient from bg to transparent) ИЛИ поменять translucent на false + добавить bg-color.

### LOW

**L1 — ZIP TextField `value` accessibility exposure**
- `placeholder="80203"` показывается accessibility-wise как `value: "80203"` даже когда поле пустое. VoiceOver озвучит "ZIP code, 80203" вместо "ZIP code, empty".
- **Фикс:** `accessibilityValue={{ text: zip || 'empty' }}` ИЛИ уберите placeholder и сделайте inline hint-label.

**L2 — Low-contrast progress % text** на Projects tab cards ("35%", "15%" рядом с severity chips).
- Текст очень светло-серый на тёмно-сером card, contrast < 3:1.
- **Фикс:** использовать `colors.textSecondary` вместо `textDim` / `textTertiary`.

**L3 — Truncated "Roof Inspection Recommended" body** на Systems dashboard без expand.
- Текст обрывается после "prevent structural..." — нет chevron или "read more". Карточка tappable (`accessibilityLabel: "Roof inspection recommended"`), но unclear.
- **Фикс:** явная chevron-стрелка справа И/ИЛИ убрать truncation если карточка крупная.
